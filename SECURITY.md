# Security policy

## Supported versions

Security fixes target the latest commit on the default branch and the latest tagged release.

## Reporting

Please report a vulnerability privately through GitHub's **Security → Report a vulnerability** feature. If private reporting is unavailable, contact the repository owner through the private address listed on their GitHub profile. Do not include tokens, exploit traffic, personal data or sensitive infrastructure details in a public issue.

Include affected version/commit, prerequisites, reproduction, impact and a suggested mitigation if known. Expect acknowledgment within 7 days and a status update within 14 days; timelines may change with severity and maintainer availability.

## Scope

Authentication bypass, denial of service, Origin/rate-limit bypass, audit tampering, secret disclosure and a path from simulated to real execution are in scope. Market-price inaccuracies and synthetic demo profitability are not vulnerabilities unless they cross the documented safety or labeling boundary.

Never test against a public deployment destructively or against third-party exchanges. Use a local instance and synthetic data.
